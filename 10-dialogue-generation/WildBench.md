## Overview

WildBench is an LLM evaluation benchmark based on real user queries. It selects 1,024 challenging tasks from over 1 million real user-model conversations (WildChat dataset), covering 12 categories. Unlike synthetic questions, WildBench's questions come directly from real user interactions with chatbots, making them more representative of actual usage scenarios.

Core innovations:
- **Real data source**: Questions come from real user conversations in the WildChat dataset, not artificially written
- **Difficulty filtering**: Uses GPT-4-Turbo, Claude-3-Sonnet, and Opus to annotate difficulty (5-point scale), retaining only medium-to-high difficulty questions
- **Length bias mitigation**: Converts "slightly better/slightly worse" judgments to "tie" to reduce the impact of model response length on scoring
- **High correlation with human preferences**: WB-Reward achieves a Pearson correlation coefficient of 0.984 with Chatbot Arena Elo (Top-6 models)

## Dataset Composition

### Data Source and Filtering Pipeline

1. **Raw data**: Over 1 million real user-model conversations from the WildChat dataset
2. **Filtering**: Remove overly short/long queries, non-English content, harmful content, near-duplicate content (cosine similarity > 0.9, using SentenceBERT)
3. **Difficulty annotation**: GPT-4-Turbo, Claude-3-Sonnet, and Claude-3-Opus each score questions, excluding "easy" questions
4. **Human verification**: Final 1,024 tasks verified by human reviewers
5. **Leak prevention**: Coordinated with the WildChat team to ensure evaluation questions do not appear in public WildChat releases

## Evaluation Method

WildBench provides two scoring metrics:

### WB-Reward (Pairwise Comparison)

| Feature | Description |
|------|------|
| Method | Pairwise comparison with 3 baseline models (GPT-4-Turbo, Claude-3-Haiku, Llama-2-70B-chat) |
| Scoring dimensions | 5 levels: much better / better / tie / worse / much worse |
| Cost | Approximately 3-4 times that of WB-Score |
| Correlation with Arena | Pearson 0.984 (Top-6), 0.973 (all 14 models) |

### WB-Score (Independent Scoring)

| Feature | Description |
|------|------|
| Method | 1-10 independent scoring, rescaled as (S-5)*2 |
| Cost | ~$5/model, ~10 minutes |
| Correlation with Arena | Pearson 0.955 (Top-6), 0.940 (all 14 models) |

### Common Features

- **Task-specific checklists**: Each question is equipped with 5-10 checklist items (generated by GPT-4-Turbo + Claude-3-Opus, human-reviewed)
- **Chain-of-Thought judging**: Requires the judge model to output reasoning process before giving a verdict
- **Length bias mitigation**: When the "winning" response is more than 500 characters longer than the "losing" response, "slightly better/slightly worse" judgments are converted to "tie"

## Example Questions

### Programming & Debugging
```
I have a Python script that processes CSV files, but it's running very slowly
on large files (over 1GB). The script uses pandas read_csv and then applies
several row-wise operations. How can I optimize this to handle large files
more efficiently?
```

### Creative Writing
```
Write a short story (500 words) told from the perspective of a lighthouse
keeper who has been alone for 20 years, when one night they see a ship
approaching that shouldn't exist — it's the same ship that crashed on the
rocks two decades ago.
```

### Reasoning
```
Three friends — Alice, Bob, and Carol — each have a different colored hat
(red, blue, or green). They can see each other's hats but not their own.
Alice says: "I don't know my hat color." Bob says: "I don't know my hat
color either." Can Carol determine her hat color? Explain your reasoning.
```

## Limitations and Controversies

1. **English-centric**: The current version only covers English queries; multilingual evaluation capability is insufficient
2. **LLM judging bias**: Although CoT judging and length bias mitigation are used, LLM-as-Judge may still have position bias and self-preference
3. **Cost and speed**: WB-Reward evaluation of a single model costs approximately $15-20, more expensive than MixEval-Hard ($0.60)
4. **Static evaluation set**: Despite having V1 and V2 versions, the question set does not dynamically update like MixEval, creating data contamination risk
5. **Single-turn evaluation**: Primarily evaluates single-turn dialogue quality, not covering multi-turn interaction capabilities
6. **Checklist generation**: Task checklists are generated by LLMs, which may introduce judging bias
