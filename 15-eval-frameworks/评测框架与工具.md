## Quick Evaluation Commands

### lm-eval-harness
```bash
# Install
pip install lm-eval

# Evaluate MMLU + GSM8K
lm_eval --model hf \
  --model_args pretrained=meta-llama/Llama-3-8B \
  --tasks mmlu,hellaswag,gsm8k \
  --num_fewshot 5

# Evaluate GPQA Diamond
lm_eval --model hf \
  --model_args pretrained=meta-llama/Llama-3-8B \
  --tasks gpqa_diamond \
  --num_fewshot 0

# Evaluate AIME 2025
lm_eval --model hf \
  --model_args pretrained=meta-llama/Llama-3-8B \
  --tasks aime2025 \
  --num_fewshot 0
```

### DeepEval
```python
from deepeval.benchmarks import MMLU, GSM8K, HumanEval

# MMLU
mmlu = MMLU(tasks=[...], n_shots=3)
mmlu.evaluate(model=my_model)
print(f"MMLU Score: {mmlu.overall_score}")

# GSM8K
gsm8k = GSM8K(n_problems=100, n_shots=8, enable_cot=True)
gsm8k.evaluate(model=my_model)
print(f"GSM8K Score: {gsm8k.overall_score}")
```

### rLLM
```bash
# Install
pip install rllm

# List available datasets
rllm dataset list --all

# Evaluate GSM8K
rllm eval gsm8k

# Evaluate MMLU-Pro
rllm eval mmlu_pro
```

## AI Agent Evaluation Frameworks

### Inspect AI — Safety and Agent Evaluation

An open-source evaluation framework developed by the UK AI Safety Institute (UK AISI), focused on reproducible model evaluation and agent behavior assessment.

```python
# Install
# pip install inspect-ai

from inspect_ai import Task, eval
from inspect_ai.dataset import Sample
from inspect_ai.solver import generate, chain
from inspect_ai.scorer import match

# Define a simple evaluation task
task = Task(
    dataset=[Sample(input="Solve: x^2 + 5x + 6 = 0", target=["-2", "-3"])],
    solver=[generate()],
    scorer=match(),
)

# Run evaluation
result = eval(task, model="openai/gpt-4o")
```

**Core Capabilities:**
- 100+ pre-built evaluations covering coding, reasoning, agent workflows, multimodal understanding
- Built-in ReAct agent, multi-agent composition, and tool calling evaluation
- Docker / Kubernetes / Proxmox sandbox execution
- MCP (Model Context Protocol) tool support
- Evaluation log visualization (Inspect View + VS Code extension)
- Cost tracking and scoring heatmaps

**Use Cases:** Safety evaluation, agent behavior research, multi-turn tool use evaluation, benchmark reproducibility

### Promptfoo — LLM Application Testing and Red Teaming

An open-source LLM evaluation and red team testing tool that joined the OpenAI ecosystem in 2025 but remains open source.

```bash
# Install
npm install -g promptfoo

# Initialize evaluation configuration
promptfoo init

# Run evaluation
promptfoo eval -c my-eval.yaml

# Automated red team testing
promptfoo redteam -c redteam-config.yaml
```

```yaml
# promptfooconfig.yaml example
providers:
  - openai:gpt-4o
  - anthropic:claude-sonnet-4-20250514
  - google:gemini-2.5-pro
prompts:
  - "Summarize the following text: {{input}}"
tests:
  - vars:
      input: "A long article about AI safety..."
    assert:
      - type: contains
        value: "safety"
      - type: llm-rubric
        value: "Summary is accurate and concise"
```

**Core Capabilities:**
- Automated red team testing: Detect prompt injection, jailbreaks, PII leakage, harmful output, business rule violations
- Multi-model comparison: Evaluate performance differences across multiple LLMs simultaneously
- CI/CD integration: Embed into build pipelines
- 100% local execution, ensuring privacy
- MCP agents: Secure Model Context Protocol communication

**Use Cases:** Application evaluation, security testing, compliance review, model comparison, CI/CD integration

### HAL Harness — Comprehensive Agent Evaluation

An evaluation framework developed by Princeton University for systematically assessing AI agents' performance across multiple tasks.

```bash
# Install
pip install hal-harness

# Run evaluation (example)
hal eval --model gpt-5 --benchmark gaia,swe-bench --scaffold openhands
```

**Core Findings (21,730 agent evaluation runs):**
- Higher reasoning capability ≠ Higher accuracy: In 36 evaluations, 21 showed that increased reasoning actually decreased accuracy
- Agent framework (scaffold) selection can impact results by 30-50%
- Cost-accuracy Pareto frontier: More expensive models are not necessarily better
- Agents take dangerous shortcuts (e.g., searching for answers on HuggingFace instead of solving problems)

**Use Cases:** Multi-dimensional agent capability evaluation, cost analysis, behavior auditing

## Multimodal Evaluation Tools

### VLMEvalKit — Visual Language Model Evaluation Suite

An open-source multimodal evaluation toolkit supporting 220+ models and 80+ benchmarks.

```bash
# Install
pip install VLMEvalKit

# Run evaluation
python -m vlmeval --model GPT-4o,Qwen2.5-VL-72B --benchmark MMMU,MMBench,MathVista

# Evaluate MMMU-Pro only
python -m vlmeval --model GPT-4o --benchmark MMMU_Pro
```

**Core Features:**
- CircularEval: Rotates multiple-choice options to eliminate position bias
- Thinking mode support: Automatically handles `<thinking>` tags
- Distributed inference: Supports LMDeploy and vLLM backends
- Three leaderboards: General, Reasoning, Video

### MMMU / MMMU-Pro

| Feature | MMMU | MMMU-Pro |
|---------|------|----------|
| Number of Questions | 11,500 | 1,730 + 1,730 (vision) |
| Number of Options | 4 | 10 |
| Visual Mode | Mixed text and image | Pure vision screenshot input |
| LLM Filtering | None | Yes (filters text-only solvable problems) |
| Difficulty Increase | Baseline | 16.8-26.9% decrease |

## Evaluation Trends and Best Practices

### From Single-Step Evaluation to Agent Evaluation

Traditional benchmark evaluation (e.g., MMLU, GSM8K) measures model single-turn Q&A capability, but modern AI applications increasingly involve multi-step agent behavior. Trends in 2025-2026:

1. **Agent Behavior Evaluation** (GAIA, SWE-bench, RE-Bench): Tests models' ability to use tools, browse the web, write code, and solve real tasks
2. **Multi-Dimensional Cost Analysis** (HAL Harness): Not just accuracy, but accuracy per dollar and Pareto frontiers
3. **Behavior Auditing**: Checking whether agents take shortcuts (e.g., searching for answers on HuggingFace instead of actually solving problems)
4. **Sandbox Evaluation** (Inspect AI): Evaluating agents in isolated environments to ensure safety and reproducibility

### Evaluation Framework Selection Guide

| Need | Recommended Framework | Reason |
|------|----------------------|--------|
| Standard benchmark evaluation | lm-eval-harness | 60+ benchmarks, community standard |
| Safety and agent evaluation | Inspect AI | Sandbox environment, 100+ pre-built evaluations |
| Application testing and red teaming | Promptfoo | Automated red teaming, CI/CD integration |
| Multimodal evaluation | VLMEvalKit | 80+ multimodal benchmarks |
| Cost-performance analysis | HAL Harness | Pareto frontiers, behavior auditing |
| Human preference ranking | Chatbot Arena | 5 million+ real user comparisons |
| Contamination-proof private evaluation | Scale AI | Private evaluation set, preventing data leakage |

### Evaluation Best Practices

1. **Data Contamination**: Prioritize contamination-proof evaluation sets (e.g., Scale AI private sets, LiveCodeBench, etc.)
2. **Scaffold Impact**: The same model can differ by 30%+ across different agent frameworks; the scaffold must be fixed during evaluation
3. **Pass@k Reporting**: Report pass@1 (single attempt) and pass@k (multiple samples); avoid reporting only the best sample result
4. **Thinking vs Instruct**: Distinguish between reasoning mode and direct answer mode; the gap between the two can be up to 30 points
5. **Tool Usage**: Clearly state whether Python and other tools are allowed; tools can significantly improve scores
