## Overview

AlpacaEval is an automated instruction following evaluation framework developed by Stanford's tatsu-lab, designed to provide a low-cost, high-efficiency way to evaluate LLM instruction following capabilities. Its core approach uses LLM-as-Judge to automatically compare model outputs against baseline outputs, thereby replacing expensive human evaluation.

AlpacaEval 1.0 was released in 2023 and quickly adopted by the community for its convenience, but exposed a severe length bias problem — longer responses were more likely to be preferred by the judge. AlpacaEval 2.0 was released in 2024, effectively mitigating this issue by introducing Length-Controlled Win Rate, significantly improving the correlation between evaluation results and human preferences.

## Dataset Composition

AlpacaEval contains approximately 800 test prompts, aggregated from multiple existing datasets:

| Data Source | Description | Approximate Prompt Count |
|---------|------|-----------------|
| AlpacaFarm | Diverse tasks from self-instruct generation | ~200 |
| Self-Instruct | Open-ended instruction generation | ~150 |
| OASST1 | Open Assistant crowdsourced data | ~100 |
| HumanEval-GPT4 | High-quality instructions generated by GPT-4 | ~100 |
| Other Sources | Programming, writing, reasoning, etc. | ~250 |

Prompt types cover a wide range, including: information QA, creative writing, code generation, text summarization, role-playing, logical reasoning, etc.

## Evaluation Method

### AlpacaEval 1.0

1. **Baseline model**: Uses text-davinci-003 outputs as the baseline
2. **Judging method**: LLM-as-Judge (default GPT-4) performs pairwise comparison between model output and baseline output
3. **Metric**: Win Rate — the proportion of model outputs preferred over the baseline
4. **Issue**: Severe length bias; longer responses were often preferred

### AlpacaEval 2.0

The core improvement in AlpacaEval 2.0 is length-controlled win rate:

1. **Baseline model**: Uses GPT-4 Turbo outputs as the baseline (stronger baseline, more discriminative)
2. **Judging method**: Also uses LLM-as-Judge, but introduces length control
3. **Core metrics**:

| Metric | Description |
|------|------|
| LC Win Rate | Length-controlled win rate, eliminating length bias |
| Raw Win Rate | Raw win rate (retained for reference) |
| Length | Average token length of model output |

4. **Length control method**: Control for response length is incorporated into the judging prompt, or statistical correction is applied to the impact of length on win rate, ensuring the win rate reflects response quality rather than length

### Judge Model Selection

AlpacaEval supports multiple judge models:

| Judge Model | Agreement with Humans | Cost | Recommended Use Case |

## Example Questions

### Information QA
```
Explain the concept of quantum entanglement to someone with a basic physics background. Include a real-world analogy.
```

### Creative Writing
```
Write a short story (200-300 words) about an AI that discovers it can dream. The story should have a surprising twist ending.
```

### Code Generation
```
Write a Python function that implements the Aho-Corasick string matching algorithm. Include docstrings and type hints.
```

### Practical Task
```
Create a weekly meal plan for a vegetarian who is trying to increase their protein intake. Include breakfast, lunch, and dinner for each day.
```

### Reasoning Analysis
```
Analyze the pros and cons of remote work vs. office work, considering both employee and employer perspectives. Present your analysis in a balanced way.
```

## Variants and Related Benchmarks

### AlpacaEval 1.0

- Predecessor to AlpacaEval 2.0
- Uses text-davinci-003 as baseline
- Has severe length bias (longer responses inflate win rates)
- No longer recommended after 2.0 was released

### AlpacaFarm

- Predecessor project to AlpacaEval
- Focused on evaluating RLHF / RLHAIR training methods
- Provides a low-cost alternative to simulated human feedback

### Related Benchmarks

- **MT-Bench**: Also an LLM-as-Judge evaluation, but focused on multi-turn dialogue quality (see [MT-Bench](MT-Bench.md))
- **IFEval**: Tests instruction following with format constraints, complementary to AlpacaEval's open-ended evaluation (see [IFEval](IFEval.md))
- **Chatbot Arena**: Human preference evaluation, can serve as a validation baseline for AlpacaEval's automated evaluation (see [Chatbot Arena](../10-dialogue-generation/Chatbot-Arena.md))

## Limitations and Controversies

1. **Length bias**: AlpacaEval 1.0's length bias has been widely discussed; 2.0 significantly improved this, but length bias cannot be fully eliminated
2. **Judge model dependency**: Evaluation results are highly dependent on the choice of judge model; different judges may produce different rankings
3. **Baseline selection**: 2.0 uses GPT-4 Turbo as baseline, which may gradually lose discriminative power as models improve
4. **Single-turn limitation**: Only evaluates single-turn instruction following, unable to reflect multi-turn dialogue capabilities
5. **Data leakage**: ~800 prompts are publicly available, creating a risk of data contamination
6. **Gap with human preferences**: Although correlation is high, LC Win Rate still differs from true human preferences
7. **English-centric**: Primarily English prompts, insufficient coverage of Chinese evaluation
8. **Style bias**: LLM judges may prefer specific writing styles (e.g., more formal, more structured responses)
